[ARQUITECTURA DE XARXA I MAQUINARI]
Els sensors hiperespectrals estan connectats via adaptadors USB a Ethernet independents. El sensor VIS té la IP 193.167.0.30 i el NIR la 193.167.0.31. Hem configurat rutes d'IP i adaptadors específics a Linux per evitar conflictes d'enrutament en la mateixa subxarxa.
Protocol TCP: És molt estricte i requereix un keepalive determinista (comandament PNG cada 1 segon) gestionat internament per evitar talls de connexió o "timeouts" (com el temut [Errno 32] Broken pipe o "Connection lost during read").
Les comandes han d'incloure obligatòriament la seva longitud codificada en 2 bytes Big Endian: struct.pack(">H", length). El socket timeout és de 20.0s per al VIS i 15.0s per al NIR.
Diferències crítiques de comportament entre sensors:
- El sensor NIR (.31) s'atabala si rep Pings durant una captura; és obligatori pausar el ping (pause_keepalive()) just abans de la captura i reactivar-lo (resume_keepalive()) immediatament després.
- El sensor VIS (.30) requereix que el keepalive estigui sempre actiu. Abans d'enviar qualsevol comanda de captura, s'ha de cridar wait_after_ping(0.4) per evitar que la comanda xoqui amb un ping en vol.
Valors MTI (Integration Time) validats experimentalment:
- VIS MTI òptim: 140.000 µs (zona lineal pura, sense artefactes espectrals al rang 700-793 nm).
- VIS MTI sub-exposat: 80.000 µs (pics espuris a 700-793 nm; ÚNIC ÚS: captura de bracketting d'entrenament).
- VIS MTI Soft Ceiling moderat: 200.000 µs (lleuger clip als materials molt foscos/absorbents; per bracketting).
- VIS MTI Hard Ceiling prohibit: >210.000 µs (saturació severa; descartar tot espectre).
- NIR MTI sweet spot: 23.750 µs. NIR MTR: 30.000 µs.
- La distància focal obligatoria validada (Knife-Edge Test) és Z = 30mm.

[ARQUITECTURA DE PROGRAMARI I NODES]
Fem servir un paquet anomenat navi_wall_interfaces (o integrat a arm_control) construït amb ament_cmake que conté els nostres serveis personalitzats .srv (HyperspectralCommand, HyperspectralConfig i PredictMaterial). La lògica recau en 3 nodes de Python:

hyperspectral_node.py: Es comunica amb el maquinari via LenzClient (TCP). Fa les calibracions obligatòries per terminal: GDS (corrent fosc/soroll), GRF (referència blanca) i GSM (mostra). Paràmetres per defecte validats: VIS MTI=140000, VIS MTR=200000, NIR MTI=23750, NIR MTR=30000. Implementa reintents (3 intents) amb validació _is_valid_frame() per a GDS i GRF. La mesura GSM té retry automàtic de 3 intents amb reinici del dict results = {} a cada intent (CRÍTIC: evita mesclar frames de captures d'intents separats). El temps d'espera adaptatiu per a l'adquisició és: max(vis_mti, nir_mti) / 1_000_000.0 + 0.5 segons.

ml_inference_node.py: Node servidor per a la inferència (Pipeline 11). Rep espectres de reflectança normalitzats i retorna el material predit amb la seva confiança. El pipeline intern del model .pkl aplica: Clamping → Trimming (màscara 975-1005 nm del gap físic sensor, retall 10 nm als extrems) → SNV (Standard Normal Variate) → StandardScaler → RandomForest(class_weight='balanced'). Llindar de rebuig estàtic: REJECTION_THRESHOLD = 0.85. Prediccions amb confiança inferior retornen "REBUTJAT_BAIXA_CONFIANCA". Si no troba el model .pkl, activa el mode Mock per a proves de xarxa.

inspection_manager.py: Client orquestrador (no toca mai la xarxa TCP). Demana la captura al node 1, calcula la reflectança i la envia al node 2. Implementa multi-capture robusta (5 captures per defecte, mediana, mínim 60% vàlides). El filtre d'estabilitat (_check_stability) analitza les bandes VIS[40:-20] i NIR[20:-20] per evitar zones UV amb denominador quasi-zero.

Càlcul de Reflectança (FÓRMULA OFICIAL):
  Reflectança = (GSM - GDS) / (GRF - GDS)
  - Implementació: np.divide(gsm_net, grf_net, out=np.zeros(256), where=grf_net > 0)
  - PROHIBIT retallar valors negatius amb np.clip. Els valors negatius (soroll tèrmic) han de mantenir-se per al correcte funcionament de StandardScaler dins del pipeline.
  - La referència blanca (paper blanc) absorbeix IR → produeix reflectança > 1.0 per a materials secs. Això és correcte i intencional; el model ML ho aprèn.

[PROTOCOL MTI BRACKETING (per a dataset d'entrenament robust)]
Per obtenir un model que generalitzi a variacions d'exposició, el protocol de captura d'entrenament fa 3 captures per mostra:
  1. VIS MTI = 80.000 µs  → sub-exposat, pics espuris a 700-793 nm (el model aprèn l'artefacte)
  2. VIS MTI = 140.000 µs → zona lineal òptima
  3. VIS MTI = 200.000 µs → Soft Ceiling moderat (clip suau en materials molt foscos)
Commanda: ros2 run arm_control inspection_manager --mode collect --label NomMaterial --bracket --zone wall1

- OBLIGATORI: np.divide(..., where=(GRF - GDS) > 0) per evitar divisions per zero.
- PROHIBIT: np.clip per retallar valors negatius (trenquaria el StandardScaler del pipeline).

- Flush_buffer() obligatori després de qualsevol send_config() als sensors.
